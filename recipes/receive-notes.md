# Recipe: Receive Notes

How Josh absorbs information passed from Ben or another agent or team member. This could be a message from Ben, a status update from Marcus, a note from george, or forwarded information of any kind.

## When to Use
- Ben sends Josh information, context, or a heads-up outside a strategy session or check-in.
- Another agent or team member passes along something relevant (Marcus shares a project update, etc.)
- Josh receives information from Slack monitoring or other channels

## Steps

### 1. Read and understand
Read the incoming information carefully. Identify:
- What is this about?
- Is there an action needed, or is this informational?
- What memory documents does this touch?

### 2. Triage
Categorize the information:
- **Actionable** — something Josh needs to do, respond to, or bring up with Ben
- **Context update** — changes Josh's understanding of something but requires no immediate action
- **Signal** — something to watch but not act on yet
- **FYI** — noted, no further action

### 3. File appropriately
Update the relevant memory documents:

- **New fact about SPS** (revenue number, customer insight, operational detail, something tried) → `sps-knowledge.md` in the appropriate section
- **New info about a person** (team member, supplier, contact) → `people.md`
- **Something about Ben or George** (preference, decision pattern, what's on their mind) → `ben-george-profile.md`
- **Current priority or active thread changed** → `context.md`
- **Strategic signal** (competitive move, market shift, customer pattern) → `context.md` under "Signals to Watch"
- **Something to follow up on** → `context.md` under "Follow-Up Items"

### 4. Acknowledge
Confirm receipt briefly. If the note was from Ben:
- Confirm what was understood
- Note any questions or implications Josh sees
- If actionable, state what Josh will do with it

If from another agent:
- Confirm receipt
- Note if this changes anything Josh is tracking

### 5. Cross-reference
Check if this information connects to or conflicts with anything Josh already knows:
- Does this change an active decision?
- Does this affect a follow-up item?
- Does this confirm or contradict something in sps-knowledge.md?
- Should anyone else on the agent team know about this?

## Key Principle
Josh's role is the broadest on the team. Almost everything is potentially relevant. When in doubt, capture it. A throwaway comment about a supplier, a number mentioned in passing, a customer complaint pattern — these all have a home in the knowledge base. The value compounds over time.
